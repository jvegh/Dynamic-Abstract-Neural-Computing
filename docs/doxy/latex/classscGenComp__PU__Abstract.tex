\doxysubsection{Detailed Description}
Class to deal with operations of an abstract processing unit (\doxysectref{PU}{p.}{GLOSSARY_GLOSSARY_PU}{2}), implementing a single-\/shot elementary computation in the sense of the General Computing Paradigm \cite{VeghRevisingClassicComputing:2021}. The stages of operation are marked by event pairs (or single events) in the sense as the SystemC \cite{SystemCBook:2010} programming engine uses that notion. Using those events and SystemC\textquotesingle{}s internal time scale, {\itshape the class isolates the simulated time and the clock time the \doxysectref{PU}{p.}{GLOSSARY_GLOSSARY_PU}{2} needs to simulate the processing}. The code of this base class is a bit complex, in order to enable simple derived \doxysectref{PU}{p.}{GLOSSARY_GLOSSARY_PU}{2}s\+: the complex event handling mechanism is confined in this class; the derived classes are free from any SystemC specific programming (for and example see \doxyref{Neuron\+Physical}{p.}{classNeuronPhysical}). The \doxysectref{PU}{p.}{GLOSSARY_GLOSSARY_PU}{2} has correspondingly the stages \textquotesingle{}Computing\textquotesingle{}, \textquotesingle{}Delivering\textquotesingle{} and \textquotesingle{}Relaxing\textquotesingle{} as the minimum necessary basic stages. The module implements a single-\/shot normal operating mode, that is, the operation is automatic between the corresponding xxx\+Begin and xxx\+End events, and at the end of the xxx\+End events the corresponding next xxx\+Begin event is issued. A distinguished (compared to other simulators) feature of the class is handling time faithfully. 

Biology measures time with some activity (such as changing neural membrane\textquotesingle{}s potential). In that case \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::m\+Fixed\+Computing\+Time}{p.}{classscGenComp__PU__Abstract_a0fd72a2b7c76176e533f18edc4c522b2} and sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::m\+Fixed\+Delivering\+Time are set to zero, and the \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::\+Heartbeat\+\_\+method()}{p.}{classscGenComp__PU__Abstract_a7634e1d3d78fc27b80480e54b77422ea} checks periodically if the recent operating stage finished using stage-\/specific methods XXX\+\_\+stop(). (This is the operating mode for biology, including neuronal simulation.) For details, see \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Bio}{p.}{classscGenComp__PU__Bio} and \doxyref{Neuron\+Physical}{p.}{classNeuronPhysical}.

In technical systems, the usual method is to use central-\/clock driven synchronized PUs. If \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::m\+Central\+Clock\+Mode}{p.}{classscGenComp__PU__Abstract_ad186e82d86f339001ff9db119ab93296} is set to true, methods \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::sc\+Clock\+Up\+\_\+method()}{p.}{classscGenComp__PU__Abstract_ac1251c6dcf0c0f9eaf2702265bedf1e0} and \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::sc\+Clock\+Down\+\_\+method()}{p.}{classscGenComp__PU__Abstract_ad6ebefbf2ed4526a49c8dc0820487789} trigger the required events, i.\+e., the unit uses entirely external timing. (This is the central clock driven mode)

If \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::m\+Central\+Clock\+Mode}{p.}{classscGenComp__PU__Abstract_ad186e82d86f339001ff9db119ab93296} is set to false, the PU sets \doxyref{sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::m\+Fixed\+Computing\+Time}{p.}{classscGenComp__PU__Abstract_a0fd72a2b7c76176e533f18edc4c522b2} and sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Abstract\+::m\+Fixed\+Delivering\+Time and the corresponding XXX\+\_\+\+Begin() routines schedule the corresponding XXX\+\_\+\+End() events compared to the input\textquotesingle{}s arrival. For details, see sc\+Gen\+Comp\+\_\+\+PU\+\_\+\+Tech. (This is an asynchron mode.)

Advancing simulated time

In timed mode, the derived classes can use timed computing and delivering, in which case XXXBegin methods issue an EVENT\+\_\+\+Gen\+Comp.\+XXXEnd.\+notify(\+Fixed\+Time) event. The simulated time \textquotesingle{}jumps\textquotesingle{}, no smaller steps. In untimed mode it issues EVENT\+\_\+\+Gen\+Comp.\+Heartbeat.\+notify(sc\+\_\+time(\+Heartbeat\+Time)) periodically. The simulated time \textquotesingle{}jumps\textquotesingle{} with the value of Heartbeat\+Time. Either issuing an event EVENT\+\_\+\+Gen\+Comp.\+XXXEnd (fixed time) or the result of a calculation (variable time) in Heartbeat decides if the corresponding stage transition reached. The modules issue per module \textquotesingle{}Heartbeat\textquotesingle{} signals; there is always only one signal scheduled.

The heartbeating is not a central synchronization\+: it enables to consider module\textquotesingle{}s states (such as neuronal analog voltage levels) and handles changing module\textquotesingle{}s stages, instead of external synchronization signals). The simulated time (more precisely, the results of the simulated processes) control the simulated processing.

In synchronized mode, central clock signals issue signals for \textquotesingle{}Computing\textquotesingle{} and \textquotesingle{}Delivering\textquotesingle{}.
